FOR COORDINATOR — Fee Router skill (FeeSplitter) — smoke deploy + publish
=========================================================================

WHAT: community skill #2 (founder's pick): a configurable N-way fee splitter.
      FeeSplitter = IMMUTABLE, pull-based, native + ERC20. NO admin, no owner,
      no pause, nothing to renounce — that's the selling point (trustless by
      construction; the ABI's only mutating fns are release/releaseAll).
      Bankr natively only splits fees inside fixed token-launch flows — this
      covers everything else (2, 3, or N recipients by basis-point shares).

      It does NOT deploy as a creation tx — it deploys THROUGH the live
      DeployerFactory (Base 0xCF4357aFdC26fa028e77291CE2F97C9dAF93F75D):
      initCode = creation-bytecode.txt ++ abi.encode(address[],uint256[])
      wrapped in deploy(bytes) (selector 0x00774360). SKILL.md carries two
      fully-encoded worked examples (2-way 50/50, 3-way 50/30/20) with the
      complete calldata ready to paste.

SMOKE DEPLOY (one, throwaway-but-real):
  node smoke-deploy-via-factory.cjs           # DRY: simulates via eth_call,
                                              #      prints predicted addr + gas
  node smoke-deploy-via-factory.cjs --live    # broadcast + verify + record
  Defaults: 50/50 between ops 0x0780…05F2 and agent 0xE2a4…aC10 (both ours —
  anything mis-sent to it stays recoverable by releasing to our own wallets).
  Env overrides: RECIPIENTS / SHARES_BPS (bps, sum 10000) / DEPLOY_RPC.
  Signer: AGENT_PRIVATE_KEY from mftusd-build/.env. Reads factory fee() and
  sends exactly that value (ctor NOT payable). Writes
  fee-splitter-smoke-deployment.json on success.

VERIFY AFTER SMOKE:
  1. eth_getCode(splitter) > 0; recipients()/shares()/recipientCount()/TOTAL_BPS
     match what you encoded (the --live path checks all of this and refuses to
     record otherwise).
  2. Send a dust native amount (e.g. 0.00001 ETH) to the splitter →
     pending(0x0, ops) == half → release(0x0, ops) pays it → repeat release
     reverts "nothing due". That's the whole lifecycle proven on mainnet.
  3. Optional Basescan source verify: single file FeeSplitter.sol, compiler
     v0.8.35+commit.47b9dedd, viaIR true, optimizer 200, evmVersion paris,
     ctor args = the ABI tail the script printed.
